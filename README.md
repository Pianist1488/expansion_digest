# Expansio Digest

Сайт дайджеста: несколько выпусков, между которыми читатель переключается прямо из шапки.

## Адреса

| Адрес | Что отдаёт |
|---|---|
| `/` | Свежий выпуск (редирект на `/digest/<id>`) |
| `/digest/10/` | Выпуск #10 |
| `/digest/09/` | Выпуск #09 |
| `/archive` | Список всех выпусков |
| `/api/digests` | JSON со списком выпусков — из него меню строит себя |

## Как добавить новый выпуск

1. Положить выпуск в `public/digests/<id>/index.html` вместе с его ассетами.
   Внутри папки пути остаются относительными (`./support.js`, `digest/fonts/...`) —
   выпуск раздаётся с адреса `/digest/<id>/`, поэтому они разрешаются сами.
2. Добавить строку в массив `digests` в начале `server.js`. Первый элемент
   массива считается свежим и открывается на главной.
3. Подключить меню: `<script src="/switcher.js" defer></script>`.

Больше ничего менять не нужно — меню и `/archive` берут список из `/api/digests`.

## Переключатель выпусков

`public/switcher.js` дописывает кнопку «Выпуски» в шапку. Он рассчитан на то, что
выпуск рисует рантайм `<x-dc>`: ждёт появления шапки внутри `#dc-root` (в исходном
шаблоне внутри `<x-dc>` лежит вторая копия шапки — вставку в неё рендер затирает)
и возвращает кнопку на место, если рантайм перерисовал шапку.

Выпуск #09 собран в один самодостаточный файл: HTML лежит внутри JSON-шаблона в
`public/digests/09/index.html` и распаковывается на клиенте. Тег скрипта там уже
вшит в шаблон. Если будешь править этот файл руками — прямые слеши в JSON-строке
должны остаться экранированными (`</script>`), иначе бандл развалится.

## Локальный запуск

```sh
npm install
npm start          # http://localhost:3000
```

Переменные окружения: `PORT`, `DATABASE_URL` (аналитика в Postgres),
`ANALYTICS_PASSWORD` (доступ к `/analytics`).

> Аналитика (`/api/track`) сейчас не собирает событий: в выпусках нет вызовов
> трекинга. Страницы `/analytics` тоже нет — роут отдаёт 404.
